Les certifications et labels permettent de structurer une démarche, de la valoriser auprès des parties prenantes, et parfois de répondre à des exigences réglementaires. Ce guide présente les principales certifications liées au numérique responsable avec des fiches détaillées pour vous aider à choisir.

**Temps de lecture estimé : 35 minutes**

### Certificat Numérique Responsable (INR)

**Organisme** : Institut du Numérique Responsable

**Description**
Certification de référence en France pour valider les compétences en numérique responsable. Couvre tous les aspects : stratégie, écoconception, achats, sensibilisation.

**Public cible**
- Responsables RSE et développement durable
- DSI et responsables IT
- Chefs de projet digitaux
- Consultants

**Prérequis**
- Aucun prérequis formel
- Connaissance de base des enjeux environnementaux recommandée
- Expérience professionnelle dans le numérique appréciée

**Contenu de la formation**

```
JOUR 1 : FONDAMENTAUX
─────────────────────
• Enjeux environnementaux du numérique
• Cycle de vie des équipements
• Indicateurs et mesure d'impact
• Cadre réglementaire (REEN, AGEC, CSRD)

JOUR 2 : MISE EN ŒUVRE
──────────────────────
• Stratégie numérique responsable
• Écoconception des services
• Achats responsables
• Sensibilisation et conduite du changement
• Plan d'action et indicateurs
```

**Compétences validées**
- Comprendre les enjeux environnementaux du numérique
- Mesurer l'empreinte d'un SI
- Définir une stratégie numérique responsable
- Mettre en œuvre des bonnes pratiques
- Accompagner le changement

**Format de l'examen**
- QCM de 60 questions
- Durée : 1h30
- Score requis : 70%
- Passage en ligne ou en présentiel

**Préparation**
1. Suivre la formation de 2 jours
2. Lire "Sobriété numérique" de Frédéric Bordage
3. Consulter les référentiels RGESN et GR491
4. Pratiquer avec les quiz en ligne

**Coût**
- Formation + certification : 1 500-2 000 €
- Repassage examen seul : 150 €
- Financement OPCO possible

**Validité et renouvellement**
- Validité : 3 ans
- Renouvellement : 50 points CPD sur 3 ans
- Formation continue ou conférences

**Ressources**
- Site : [institutnr.org](https://institutnr.org)
- MOOC préparatoire : [academie-nr.org](https://www.academie-nr.org)

**Retours d'expérience**

> "La certification m'a donné la légitimité pour porter le sujet en interne. Les 2 jours de formation sont denses mais très concrets." — Marie, DSI

> "L'examen est accessible si on a bien suivi la formation. Les questions portent autant sur les concepts que sur les cas pratiques." — Thomas, Consultant RSE

### Green Software for Practitioners

**Organisme** : Green Software Foundation / Linux Foundation

**Description**
Formation gratuite et certification internationale pour les développeurs et professionnels IT souhaitant créer des logiciels plus durables.

**Public cible**
- Développeurs (tous langages)
- Architectes logiciels
- DevOps / SRE
- Product managers

**Prérequis**
- Connaissances de base en développement logiciel
- Anglais (formation en anglais)

**Contenu de la formation**

```
MODULE 1 : INTRODUCTION
───────────────────────
• Définitions et concepts clés
• Pourquoi le logiciel durable ?
• Green Software Foundation

MODULE 2 : EFFICACITÉ CARBONE
─────────────────────────────
• Mesurer les émissions
• Sources d'émissions du logiciel
• Intensité carbone

MODULE 3 : EFFICACITÉ ÉNERGÉTIQUE
─────────────────────────────────
• Consommation des composants
• Optimisation algorithmique
• Idle vs charge

MODULE 4 : CONSCIENCE CARBONE
─────────────────────────────
• Intensité carbone du réseau
• Demand shifting
• Demand shaping

MODULE 5 : EFFICACITÉ HARDWARE
──────────────────────────────
• Carbone embarqué
• Amortissement
• Prolongation de vie

MODULE 6 : MESURE
─────────────────
• Métriques standards
• SCI (Software Carbon Intensity)
• Outils de mesure

MODULE 7 : STANDARDS
────────────────────
• SCI Specification

**Format de l'examen**
- QCM de 40 questions

**Coût**
- Formation : Gratuite

- Badge Credly officiel

**Préparation**
1. Suivre le cours en ligne (8h)
2. Pratiquer les quiz de fin de module
3. Lire la documentation SCI
4. Explorer les Green Software Patterns

**Ressources**
- Formation : [learn.greensoftware.foundation](https://learn.greensoftware.foundation)
- Patterns : [patterns.greensoftware.foundation](https://patterns.greensoftware.foundation)

**Retours d'expérience**

### Éco-conception des Services Numériques

**Organisme** : GreenIT.fr / Collectif Conception Numérique Responsable

**Description**
Formation et certification spécialisée sur l'éco-conception web et mobile, avec un focus pratique sur les bonnes pratiques de développement.

**Public cible**
- Développeurs frontend et backend
- Intégrateurs web
- UX/UI designers
- Chefs de projet web

**Prérequis**
- Expérience en développement web
- Connaissance HTML/CSS/JS de base

**Contenu de la formation**

```
JOUR 1 : FONDAMENTAUX ET MESURE
───────────────────────────────
Matin :
• Impact environnemental du web
• Cycle de vie d'une page web
• Référentiel RGESN
• Méthodologie d'audit

Après-midi :
• Outils de mesure (EcoIndex, Lighthouse)
• Audit pratique d'un site
• Interprétation des résultats
• Exercices

JOUR 2 : OPTIMISATION
─────────────────────
Matin :
• Optimisation des images et médias
• CSS et JavaScript sobres
• Performance et caching
• Hébergement responsable

Après-midi :
• Intégration en projet agile
• Checklists et DoD
• Cas pratiques
• Passage de la certification
```

**Compétences validées**
- Auditer l'empreinte d'un site web
- Appliquer les bonnes pratiques d'écoconception
- Optimiser les médias et le code
- Intégrer l'écoconception dans un projet

**Format de l'examen**
- QCM de 50 questions
- Durée : 1h
- Score requis : 65%
- En présentiel ou en ligne

**Coût**
- Formation + certification : 1 200-1 500 €
- Certification seule : 200 €
- Financement OPCO possible

**Ressources**
- Formation : [greenit.fr/formations](https://www.greenit.fr/)
- Référentiel : [gr491.isit-europe.org](https://gr491.isit-europe.org)
- Bonnes pratiques : [collectif.greenit.fr](https://collectif.greenit.fr)

### FinOps Certified Practitioner

**Organisme** : FinOps Foundation

**Description**
Certification FinOps avec une dimension GreenOps de plus en plus présente. Utile pour optimiser coûts et empreinte carbone du cloud.

**Public cible**
- Cloud engineers
- DevOps / SRE
- Finance IT
- Architectes cloud

**Prérequis**
- Expérience cloud (AWS, GCP, Azure)
- Notions de coûts cloud

**Contenu**

```
DOMAINES COUVERTS
─────────────────
• Principes FinOps
• Personas et collaboration
• Anatomie de la facture cloud
• Allocation et showback
• Optimisation (right-sizing, reservations)
• Unit economics
• KPIs et métriques
• GreenOps et durabilité (nouveau module)
```

**Lien coûts/carbone**
- Les optimisations FinOps réduisent souvent l'empreinte carbone
- Right-sizing = moins de ressources = moins d'émissions
- Scheduling = usage intelligent = moins de gaspillage

**Format de l'examen**
- 50 questions
- Durée : 1h
- Score requis : 75%
- En ligne, surveillé

**Coût**
- Formation : Variable (500-2000 $)
- Examen seul : 300 $

**Ressources**
- Site : [finops.org](https://www.finops.org)
- Certification : [learn.finops.org](https://learn.finops.org)

**Description**
Certification globale pour les entreprises à impact positif. Inclut des critères environnementaux applicables au numérique.

```
GOUVERNANCE (max 22 points)
───────────────────────────
• Mission et engagement
• Éthique et transparence
• Parties prenantes

COLLABORATEURS (max 53 points)
──────────────────────────────
• Rémunération et avantages
• Formation et développement
• Santé et sécurité
• Satisfaction

COMMUNAUTÉ (max 44 points)
──────────────────────────
• Diversité et inclusion
• Impact économique local
• Engagement civique
• Fournisseurs

ENVIRONNEMENT (max 44 points) ← Pertinent pour le numérique
──────────────────────────────
• Management environnemental
• Émissions de GES
• Gestion des déchets
• Utilisation des ressources

CLIENTS (max 44 points)
───────────────────────
• Impact des produits/services
• Pratiques marketing

**Score requis** : 80 points sur 200

**Application au numérique**

4. **Certification** : Engagement légal + paiement
5. **Recertification** : Tous les 3 ans

**Coût**
- Basé sur le chiffre d'affaires annuel
- De 1 000 € à 50 000 €/an

**Ressources**

## 4. Normes ISO

### ISO 14001 - Management environnemental

**Description**
Norme internationale pour les systèmes de management environnemental (SME). Cadre structurant pour gérer les aspects environnementaux d'une organisation.

**Application au numérique**

| Exigence ISO 14001 | Application IT |
|-------------------|----------------|
| Contexte de l'organisme | Cartographie des enjeux numériques |
| Aspects environnementaux | Inventaire équipements, énergie, déchets |
| Exigences légales | DEEE, AGEC, REEN |
| Actions | Prolongation durée vie, écoconception |
| Compétences | Formation Green IT |
| Surveillance | Indicateurs (PUE, empreinte carbone) |
| Audit interne | Revue des pratiques IT |

**Certification**
- Audit par organisme accrédité (AFNOR, Bureau Veritas, etc.)
- Audit de certification (initial)
- Audits de surveillance (annuels)
- Audit de renouvellement (3 ans)

---

### ISO 50001 - Management de l'énergie

**Description**
Norme pour les systèmes de management de l'énergie. Particulièrement pertinente pour les data centers et grandes infrastructures IT.

**Application aux data centers**

```
EXIGENCES ISO 50001 - DATA CENTER
─────────────────────────────────
• Politique énergétique
• Revue énergétique (consommations, usages)
• Indicateurs de performance (PUE, ERE, WUE)
• Objectifs et cibles d'économie
• Plan d'action énergétique
• Compétences et sensibilisation
• Surveillance et mesure
• Audit énergétique
• Amélioration continue
```

**Indicateurs typiques**

| Indicateur | Formule | Cible |
|------------|---------|-------|
| PUE | Énergie totale / Énergie IT | < 1,4 |
| ERE | Énergie réutilisée / Énergie IT | > 0,5 |
| WUE | Eau / Énergie IT | < 1,8 L/kWh |

## 6. Comment Choisir

### Matrice de décision

```
┌─────────────────────────────────────────────────────────────────┐
│                    MATRICE DE DÉCISION                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  OBJECTIF                         CERTIFICATION RECOMMANDÉE     │
│  ─────────────────────────────    ─────────────────────────     │
│                                                                 │
│  Monter en compétences (tech)     Green Software + Écoconception│
│  Monter en compétences (général)  Certificat NR (INR)           │
│  Valoriser l'entreprise (FR)      Label NR                      │
│  Valoriser l'entreprise (INT)     B Corp ou EcoVadis            │
│  Répondre aux AO publics          Label NR + EPEAT fournisseurs │
│  Gérer un data center             ISO 50001 + Blue Angel        │
│  Structurer la démarche RSE       ISO 14001                     │
│  Optimiser les coûts cloud        FinOps + Green Software       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### Questions à se poser

**Pour une certification individuelle :**
1. Quel est mon profil (technique, management, RSE) ?
2. Quel budget puis-je mobiliser ?
3. La certification est-elle reconnue dans mon secteur ?
4. Quels sont les prérequis ?
5. Comment maintenir la certification dans le temps ?

**Pour un label organisationnel :**
1. Quelle est notre maturité actuelle ?
2. Avons-nous le sponsoring de la direction ?
3. Quelles ressources pouvons-nous mobiliser ?
4. Le label est-il demandé par nos clients/partenaires ?
5. Quel ROI attendons-nous ?

### Pièges à éviter

```
┌─────────────────────────────────────────────────────────────────┐
│                    PIÈGES À ÉVITER                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ❌ Certification sans action                                   │
│     → Le label sans changement réel = greenwashing              │
│                                                                 │
│  ❌ Multiplicité des labels                                     │
│     → Mieux vaut un label bien maintenu que plusieurs bâclés   │
│                                                                 │
│  ❌ Focus uniquement sur le certificat                          │
│     → L'objectif est la compétence, pas le papier              │
│                                                                 │
│  ❌ Sous-estimer l'effort de maintien                           │
│     → Renouvellement et amélioration continue nécessaires       │
│                                                                 │
│  ❌ Choisir sans consulter les parties prenantes                │
│     → Le label doit répondre à un besoin réel                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## Conclusion

Il n'existe pas de certification universelle pour le Green IT. Le choix dépend de votre profil, vos objectifs et votre contexte. L'essentiel est de :

1. **Commencer quelque part** : Une certification imparfaite vaut mieux que pas de démarche
2. **Progresser** : Utilisez la certification comme point de départ, pas d'arrivée
3. **Agir** : Le label doit refléter des actions réelles
4. **Communiquer** : Partagez vos apprentissages et vos progrès

Les certifications évoluent régulièrement. Cette liste sera mise à jour pour refléter les changements du domaine.
